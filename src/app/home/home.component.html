Routing

Routing można bardzo prosto skonfigurować. Wystarczy przy tworzeniu projektu angularowego stworzyć z routingiem a następnie dodać w module app-routing odpowiedni obiekt js (zobacz moduł)

Links

Aby linki działały trzeba dodać dyrektywe routerLink=url. Url musi wskazywać ścieżke rekatywną do naszego komponentu. Możemy również podać tablicę wtedy weźmie pierwszy element który pasuje. Trzeba pamiętać, że jest różnica między /route a route. To pierwsze to absolutna ścieżka i działa ona w taki sposób, że dokleja adres do naszego głównego adresu strony internetowej, natomiast drugie to relatywna i dokleja ten adres do adresu na którym jesteśmy

Jeśli chcemy dodać jakiś cssowy styl podczas tego jak jakiś link jest aktywny używamy dyrektywy/atrybutu: routerLinkActive i podajemy w stringu jaką klase ma nałożyć bootstrap po tym jak link będzie aktywny. Problem jaki napotkamy z tym rozwiązaniem jest taki, że angular da status aktywny dla każdej podścieżki więc do tej podstawowej też. Aby tego uniknąć stosujemy routerLinkOptions (zobacz plik ts)

Route parameters and query parameters

aby przekazać route parameter musimy podać w routingu coś takiego /route/:id.

aby przekazać query parameters (czyli coś takiego: /route?query=1&second=2). Musimy zrobić coś takiego. Podajemy dyrektywe do linku o taką: qeuryParams=query: value}. Jest jeszcze coś takiego jak fragment to możęmy przekazywać przez dyrektywe fragment=''

Jeśli chcemy to zrobić za pomocą metody navigae w kodzie to poprostu dodajemy następny argument do navigate js obiekt który będzie trzymał nasze queryParams
